# Contribution Guidelines

Thank you for considering contributing to JagX-monie! We appreciate your help.

## Guidelines for Contributions
1. **Issues**: Before starting work, please check if there is an open issue that your contribution would address. You may also create a new issue to discuss your ideas.
2. **Discussions**: Engage in discussions for any major changes you plan to make.
3. **Code Review**: All contributions should be submitted via pull requests, and they will be reviewed by the maintainers.

## Code Standards
- Follow the prevailing code style in the project (check existing files for formatting standards).
- Write clear, readable code.
- Include comments where necessary for the clarity of logic or decisions.
- Ensure that all new code is covered by tests, and that tests are passing before submitting a pull request.

## Pull Request Process
1. Fork the repository and create your branch from `main`.
2. Make your changes, ensuring they are commited with clear messages.
3. Push your changes to your forked branch.
4. Submit a pull request to the `main` branch of the JagX-monie repository and ensure you have linked any relevant issues.
5. Respond to any feedback and make necessary revisions.

## Branch Naming Conventions
- Use clear and concise names for your branches. For example:  
  - `feature/your-feature-name`  
  - `bugfix/your-bugfix-name`  
  - `hotfix/your-hotfix-name`

By following these guidelines, you will help maintain the quality and organization of the project. Thank you for your contributions!
